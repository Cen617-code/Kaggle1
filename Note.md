# Kaggle 学习笔记

## 2024/4/13 — 房价预测 (Housing Prices)

### 学到的知识点

**1. 缺失值处理**
- 先分析缺失原因：有些缺失是有意义的（如没有泳池 → PoolQC 为空），不是真的数据丢失
- 高缺失率（>80%）的列可以考虑直接删除，放进模型反而增加噪声
- 数值型缺失用中位数填充，类别型缺失用众数或填 `"None"`

**2. 特征选择**
- 用 `corr()` 看各特征与目标变量的相关性，优先选高相关特征
- `OverallQual`（0.79）是最强预测因子，比 `LotArea`（0.26）有用得多

**3. 特征工程**
- 将多个相关特征组合成一个新特征，效果往往更好：
  - `TotalSF = TotalBsmtSF + 1stFlrSF + 2ndFlrSF`（总面积，相关性 0.78）
  - `HouseAge = YrSold - YearBuilt`（房龄，负相关 -0.52）
  - `TotalBath = FullBath + HalfBath + BsmtFullBath + BsmtHalfBath`（总浴室数，0.61）
- 组合后的新特征保留，被组合的旧特征可以去掉，避免冗余

**4. 模型调参**
- 用 `cross_val_score` 交叉验证来选择参数，而不是只看一次 train_test_split
- `n_estimators`：200 性价比最高，再多提升很小
- `max_depth`：10 是个好的默认值，太深容易过拟合

**5. 常见错误**
- `predict(X)` 误用了训练集特征而非 `test_X`，导致输出长度不匹配（1460 vs 1459）
- 训练集和测试集必须做相同的特征工程，否则特征不一致
